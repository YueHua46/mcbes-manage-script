# 领地内飞行：实现方案与配置设计

本文档面向 **杜绝熊孩服务器插件（mcbes-manage-script）** 当前架构，说明在 **Minecraft 基岩版 BDS + `@minecraft/server`（项目当前约 `2.8.0-beta` / `1.26.20-preview`）** 下，如何实现「仅在领地内、可全服配置、可选与经济绑定」的飞行能力，并给出**无论管理员如何配置都成立**的核心逻辑与推荐落地方式。

## 1. 目标与平台约束

### 1.1 功能目标

- **地理范围**：仅当玩家处于**某块领地 AABB 内**（与现有 `landManager.testLand` 一致）且满足权限时，才允许保持「脚本授予的飞行能力」。
- **全服开关**：服务器管理员可关闭整个功能（关闭后所有人立即按规则收回能力）。
- **经济可选**：管理员可选择是否与金币系统绑定；若绑定，可配置**每次购买/续期对应的秒数**与**每次扣费金额**。
- **安全**：不信任玩家在获得飞行后，仍不应因飞行而绕过现有领地保护（本项目领地破坏/放置等多在 `beforeEvents` 中处理，与是否飞行无直接关系）。

### 1.2 API 现实情况（必须写进实现预期）

在当前依赖的 `@minecraft/server` 类型定义中：

- `Player` 具备 `isFlying` 等读状态能力，但 **没有** 类似「单独打开生存飞行」的一等公民组件（检索 `index.d.ts` 无 `mayfly` / `EntityAbility` 一类稳定 API）。**本功能仅通过 `/ability … mayfly` 实现**（见 [§5.1](#51-使用-ability-切换-mayfly)）。
- 在**未**开启教育版世界选项时，该命令会无效；实现上必须 **检测失败并提示**（见 [§5.1.1](#511-未开教育版时的检测与用户提示)），且 **不得**在判定失败后再扣经济。

**结论**：业务上 **领地判定 + 会话 + 经济** 均围绕 **`/ability mayfly`**；全服默认 **开启** 领地飞行（`landFlightEnabled` 默认 `true`），由服主按需关闭。

## 2. 管理员可配置项（建议）

与现有 `scripts/features/system/services/setting.ts` 中 `IModules` / `defaultSetting` 的扩展方式一致，建议新增下列键（名称可按项目命名习惯微调，但语义建议保留）：

| 配置键 | 类型 | 说明 |
|--------|------|------|
| `landFlightEnabled` | `boolean` | 是否启用「领地内飞行」整套逻辑。**建议默认 `true`（开着）**；`false` 时不再授予能力，并应在短延迟内对在线玩家执行统一回收。 |
| `landFlightUseEconomy` | `boolean` | 是否在**开启飞行会话（一次购买）**时扣金币。`false` 表示仅受权限与领地范围约束，不按次扣费（仍可选是否消耗「免费时长」——建议见 [§3.2](#32-与经济模块总开关的关系)）。 |
| `landFlightSecondsPerUse` | `string`（存数字） | **每成功开启 1 次**飞行会话，给予的**有效秒数**（整数，建议下限 1，上限自行 clamp，例如 1～86400）。 |
| `landFlightGoldPerUse` | `string`（存数字） | 当 `landFlightUseEconomy === true` 且经济系统视为可用时，每次开启扣除的金币数量（非负整数；0 表示形式上绑定经济但实际免费）。 |

可选扩展（非本次硬性需求，但常见服主会问）：

- `landFlightAllowStack`：是否允许在会话未结束时再次购买**叠加**剩余时间（`true` 则 `expireAt += seconds`，`false` 则拒绝或仅刷新为更长一档——需在 UI 说明）。
- `landFlightCooldownSec`：两次购买之间的冷却，防连点。

以上配置应通过现有 **`/yuehua:setting`** 与 **系统设置表单**（`scripts/ui/forms/system/`）暴露，并在 `command.ts` 的 `settingDescriptions` 中补充说明，风格与 `pvpSeizeAmount`、`redPacketExpiryHours` 等一致。

## 3. 「一次飞行」的语义（秒数与扣费）

### 3.1 推荐语义（与需求对齐）

- **1 次**：指玩家触发一次「开启领地飞行会话」的成功操作（菜单按钮或命令）。
- **效果**：从成功时刻起，在 **T = `landFlightSecondsPerUse` 秒**内，只要仍满足 [§6](#6-核心状态机与强制回收条件) 的保持条件，则维持 **`/ability` 授予的 mayfly**。
- **扣费**：若 `landFlightUseEconomy` 为真，则在**本次成功开启前**做原子检查：`hasEnoughGold` → `removeGold`；扣费失败则不开启会话、不授予能力。
- **与 `/ability` 的衔接**：「成功开启」应包含 **存档已支持 `/ability`（教育版选项已开）**；若环境检测失败（[§5.1.1](#511-未开教育版时的检测与用户提示)），**不得扣费**、不进入会话。

> 说明：用「到期时间戳 `expireAt`（毫秒）」比「剩余 tick 数」更直观，且跨卡顿/低 TPS 时误差更小。

### 3.2 与经济模块总开关的关系

项目里经济是否启用通常由 `economy` 模块开关及 `Economic.isEconomyEnabled()` 等逻辑共同决定（见 `scripts/features/economic/services/economic.ts`）。建议约定：

- **`landFlightEnabled === false`**：整个功能关闭，不谈扣费。
- **`landFlightUseEconomy === false`**：永不因本功能扣费；会话仍可按 `landFlightSecondsPerUse` 限时（或你另行定义「免费不限时」——不推荐，易引发纠纷）。
- **`landFlightUseEconomy === true` 但经济全局关闭**：应在文档与 UI 中明确一种策略，例如：
  - **推荐**：视为「无法扣费则禁止付费开启」，并提示管理员打开经济或关闭经济绑定；
  - **备选**：退化为免费限时。  
  实现时择一并写死，避免玩家困惑。

## 4. 谁能使用领地飞行

与现有领地「管理/信任」语义对齐，可降低沟通成本：

| 角色 | 是否默认可使用 |
|------|----------------|
| 服务器管理员（现有 `isAdmin`） | 可实现为**始终可飞**或与普通玩家一致，由你方产品决定；若「始终可飞」，建议**不走经济扣费**以免误扣。 |
| 领地主人、`members`、公会领地同公会成员（`landManager.isPlayerTrustedOnLand`） | **建议默认可使用**（与传送、领地管理入口的信任模型一致）。 |
| 仅进入公开领地、但无信任关系的玩家 | **默认不建议**开放；若服主要求「游客也可买飞」，应单开配置项并明确与安全、骚扰风险，此处不展开。 |

**默认推荐**：与 `teleportToLand` 的权限思路类似——**领主 / 信任 / 公会成员**才允许发起「开启会话」；管理员可走单独分支。

## 5. 基岩版上「飞行」的技术实现

### 5.0 `/ability mayfly` 的环境前提（教育版 + 作弊）

在 **Minecraft 基岩版**（含 BDS）中，`/ability` 属于**教育版相关能力**，与 Wiki 及社区经验一致：**通常需要同时满足**：

1. **世界开启「教育版 / Education」选项**（创建世界或编辑世界时的开关；BDS 侧需在世界存档或 `server.properties` / 官方文档允许的方式中启用教育功能，具体以你使用的 BDS 版本说明为准）。
2. **允许作弊（Cheats on）** 或等价的管理权限，使命令可被解析与执行。

若**未**开启教育版相关选项，即使开了作弊，`/ability … mayfly` 也可能**不可用、无效果或被拒绝**。因此：

- **不能**假定所有存档均可飞；必须在玩家使用时做**失败兼容**（见 [§5.1.1](#511-未开教育版时的检测与用户提示)），且**扣费不得早于「确认可飞」**（先探测 `/ability` 是否对该环境生效，再 `removeGold`；探测失败则整单取消）。
- 若服主**长期**不愿开启教育版，只能通过 **`landFlightEnabled: false`** 关闭本功能，避免玩家反复看到失败提示。

### 5.1 使用 `/ability` 切换 mayfly

在会话生效且玩家处于「允许保持飞行」的状态时：

- 执行（示例，需按你服命名与选择器调整）：
  - 开启：`/ability <玩家> mayfly true`
  - 关闭：`/ability <玩家> mayfly false`

在脚本中可优先考虑：

- `player.runCommand("ability @s mayfly true")`（与项目内已有 `runCommand` 用法一致，见 `land-manager.ts`、`waypoint.ts` 等）；或
- `dimension.runCommand(...)` 带明确玩家名，避免 `@s` 上下文歧义。

**前提**：见 [§5.0](#50-ability-mayfly-的环境前提教育版--作弊)。若命令静默失败，须有**可观测日志**（`console.warn`），并对玩家展示 [§5.1.1](#511-未开教育版时的检测与用户提示) 中的固定说明。

### 5.1.1 未开教育版时的检测与用户提示

**目标**：在**默认开启**领地飞行、且**仅**通过 `/ability` 实现的前提下，存档若**未**打开「Minecraft 教育版」相关世界选项，玩家点击使用时不应静默失败；应提示**服主如何在本机开启该选项**。

**检测思路（实现可择一或组合，以不刷屏为准）**：

1. **命令返回值**：若当前 `@minecraft/server` 中 `runCommand`（或等价 API）返回**失败**（如 `success === false`、错误码表明未知命令等），可**直接**判定本次授予失败，进入提示流程。
2. **执行后短延迟复核**：若 API 无法区分「命令被忽略」，可在 `system.runTimeout` 后结合玩家状态推断（例如：已在地面上、未进入可飞行状态且重复尝试仍无法起飞——具体以实测为准）；**注意**避免误判（延迟建议 1～5 tick，且同玩家冷却，防止聊天刷屏）。
3. **世界级缓存标志**（推荐）：首次失败后将 `educationFlightUnavailable` 一类标志写入**世界动态属性**（或仅内存 + 管理员重载），后续玩家直接使用缓存路径提示，减少重复 `runCommand`；服主在服务端开启教育版并重启/重载后应**清除该标志**（可由管理员菜单提供「重新检测」按钮）。

**经济顺序（重要）**：须在**确认 `/ability` 对该玩家本次生效**（或按上款判定为「环境正常」）之后，再执行 `removeGold`；若环境不可用，**不得扣费**。

**向玩家展示的固定文案（可直接用作 `sendMessage`，可按产品微调）**：

```text
当前存档未开启「Minecraft 教育版」相关功能，领地飞行无法使用。
请服主将服务器存档导出并导入到本地客户端，在编辑该世界时开启作弊，
在「无敌模式」或世界选项相关设置中，找到并打开「Minecraft Education 功能」
（部分版本在「教育」分类下，文案可能略有差异），保存世界后再将存档导回服务器端使用。
```

说明：客户端界面因版本/语言不同，菜单名称可能为「无敌模式」「作弊」「教育版」等；实现时以实机为准。**核心信息**不变：**本地打开世界编辑 → 开启作弊 → 打开 Minecraft Education 功能 → 保存后导回服务器**。

**仅管理员可见的补充行（可选）**：在 `isAdmin` 为真时额外一行，便于服主在控制台/聊天排查：「亦请确认服务器已允许作弊，且脚本日志中无 ability 相关错误。」

### 5.2 会话保持与领地边界

- **进入会话后**：每 tick（或每 N tick，例如 5～10 tick 一次以减负）检查：
  1. `landFlightEnabled`；
  2. 玩家仍在线；
  3. `now < expireAt`；
  4. `testLand` 仍在某领地内，且对该 `insideLand` 满足 [§4](#4-谁能使用领地飞行)；
- **任一不满足**：立即执行 `ability mayfly false` 并清理内存状态。

边界细节：

- **双脚中心点**：与现有 `scripts/events/handlers/land.ts` 中领地检测类似，使用玩家所在方块/位置取整规则要与 `testLand` 一致，避免「脚在界外头在界内」的争议；若需「只要在领地上空垂直柱内即算」，当前 AABB 已覆盖 Y 范围，一般无需改数学，只需统一用 `testLand` 的坐标源。

## 6. 核心状态机与强制回收条件

### 6.1 玩家运行时状态（内存）

建议 `Map<playerId, LandFlightSession>`，仅在会话进行中存在记录：

```ts
interface LandFlightSession {
  /** 会话到期时间（毫秒，Date.now()） */
  expireAt: number;
  /** 上次已同步到游戏内的「应处于飞行会话」标志，避免每 tick 重复执行 ability */
  appliedFlight: boolean;
  /** 可选：记录授予时维度，用于调试或跨维度保护 */
  dimensionId?: string;
}
```

### 6.2 状态转移（文字版）

1. **Idle** → **Active**：玩家触发开启 → 通过权限与经济校验 → 写入 `expireAt = now + seconds * 1000` → `ability mayfly true`。
2. **Active** → **Idle**（回收）：
   - 超时 `now >= expireAt`；
   - 离开所有领地（`testLand` 无领地）；
   - 仍在领地但**失去信任关系**（罕见，如被踢出公会、领地转让）；
   - `landFlightEnabled` 被管理员关闭；
   - 玩家离线、切维度（若你希望「切维度一律收回」——推荐，最简单）；
   - 玩家死亡（推荐收回，避免复活点在野外仍飘着）。

每次进入 **Idle** 时：**务必**执行 `ability mayfly false`，并 `delete` Map 项。

### 6.3 与管理员的交互

- 管理员将 `landFlightEnabled` 设为 `false` 后，下一轮全局扫描应对所有在线玩家执行 `ability mayfly false`，并清空 Map。

## 7. 与现有模块的对接点

| 模块 | 路径/符号 | 用途 |
|------|-----------|------|
| 领地检测 | `landManager.testLand(location, dimensionId)` | 是否在领地内、当前 `ILand` |
| 信任判定 | `landManager.isPlayerTrustedOnLand(land, playerName)` | 是否允许使用本功能 |
| 全服配置 | `setting.getState` / `setState`，扩展 `IModules` 与 `defaultSetting` | 开关、秒数、金币、是否绑定经济 |
| 经济扣款 | `Economic.getInstance().removeGold` / `hasEnoughGold` | 绑定经济时扣费 |
| 定时逻辑 | `system.runInterval` | 与 `scripts/events/handlers/land.ts` 类似，可用独立 interval，注意频率 |
| 事件（可选） | `world.afterEvents.playerLeave` / `playerSpawn`；`world.afterEvents.entityDie` | 清理会话、死亡收回 mayfly |

**注册位置**：建议新建 `scripts/features/land/services/land-flight.ts`（或 `scripts/features/player/...`），在 `main` 或现有 `registerLandEvents` 同级初始化函数中引用，避免把 `land.ts` 事件文件撑得过长。

## 8. 数据结构与性能

- **默认无需持久化**：飞行会话为短时在线状态，下线即清；避免写动态属性 I/O。
- **扫描频率**：不必每 tick 对所有玩家跑 `testLand`（`testLand` 会遍历领地列表）。可参考 `behavior-log.ts` 的做法使用**秒级或数 tick 级**间隔；或仅对「当前存在 `LandFlightSession` 的玩家」每 tick 检测，其他玩家零开销。
- **命令节流**：仅在 `appliedFlight` 与实际需求不一致时调用 `ability`，避免高频刷命令。

## 9. UI / 命令入口（建议）

- **领地菜单**（`scripts/ui/forms/land/index.ts`）：对可使用者展示「领地飞行：开启一次（剩余 xx 秒 / 花费 xx 金币）」。
- **命令**（可选）：例如 `/yuehua:landflight` 子命令，便于自动化测试；需接 `isAdmin` 或权限与菜单一致。

文案需说明：

- 仅在领地内生效；
- 离开领地立刻失效；
- 与经济绑定时的单价与时长；
- 若触发 [§5.1.1](#511-未开教育版时的检测与用户提示) 失败分支，展示该节中的**固定说明**（或摘要 + 详见聊天全文）。

## 10. 测试与验收清单

1. **开关**：关闭 `landFlightEnabled` 后，正在飞行的玩家落地（mayfly 关）。
2. **边界**：站在领地边界外一格不应保持飞行；从领地边缘走出应立刻收回。
3. **超时**：`landFlightSecondsPerUse` 到期后收回。
4. **经济**：金币不足拒绝开启；扣除成功与失败时钱包数额正确；`removeGold` 理由字段便于后台审计。
5. **离线/重进**：重进后不应残留 mayfly（除非新会话）。
6. **维度**：下界/末地若也允许领地，需明确是否与主世界一致；若不做跨维度延续，切维度应收回。
7. **未开教育版**：应出现 [§5.1.1](#511-未开教育版时的检测与用户提示) 中文案；**不得扣费**；缓存标志与「重新检测」行为符合预期。
8. **作弊关闭**：可与教育版缺失区分或合并为同一失败提示；日志可查。

## 11. 小结

- **业务层**：用 `ServerSetting` 提供「总开关（**建议默认开**）、是否绑定经济、每次秒数、每次金币」，并与 `Economic`、领地信任模型对齐；飞行能力**仅**通过 `/ability mayfly` 授予与收回。
- **技术层**：`@minecraft/server` 无独立 mayfly API。**`/ability mayfly` 依赖「教育版世界选项 + 作弊」**；实现必须在失败时走 [§5.1.1](#511-未开教育版时的检测与用户提示)，向玩家展示**导回本地、开作弊、开启 Minecraft Education 功能、再导回服务器**的说明，且**先确认可飞再扣费**。状态机仍为 **`expireAt` + 领地检测 + 信任检测**，退出路径对称执行 `ability mayfly false`。
- **工程层**：会话放内存 Map、定时器节流、在 `features/land` 或独立服务中实现并与现有 `landManager` / `setting` 集成；可选世界级缓存标志减少重复探测；管理员可提供「重新检测教育版能力」与关闭 `landFlightEnabled`。

本文档仅描述设计与逻辑契约；具体 PR 实现时可按 [§7](#7-与现有模块的对接点) 拆分提交。
