# 更新说明 · 2026-05-24

## 领地快照

- 新增**领地快照**功能，管理员可以在领地详情中为指定领地保存当前结构快照，并在需要时恢复或删除快照。入口：**领地详情** → **领地快照**。
- 保存快照时默认只保存方块；也可以选择**包含实体**，会把画、展示框、生物、掉落物等非玩家实体一并保存。恢复包含实体的快照时，会先清理该区域内的非玩家实体，再放回快照中的实体。
- 大领地会自动拆成多个结构分片保存/恢复，并按任务队列逐步执行。保存或恢复期间可能出现短暂卡顿，建议不要在玩家正在施工时恢复快照。
- 管理员可开启**自动保存快照**。入口：**服务器设置** → **领地系统管理** → **快照功能设置**。可设置自动保存间隔、每块领地保留多少个自动快照、是否包含实体，以及结构切块上限。
- 自动快照达到上限后，会自动删除最旧的自动快照；**手动保存的快照不会被自动覆盖**。

## 行为日志

- 新增**行为日志查询器**道具。管理员可在 **服务器设置** → **玩家行为日志** 中点击**获取日志查询器**，手持后点击方块即可查看该方块附近的行为记录，适合快速追查箱子、领地、破坏等现场问题。
- 查询器范围半径可在**行为日志监控设置**中调整，支持 0～32 格，默认 3 格。
- 行为日志新增**领地破坏尝试**记录：玩家在无权限领地里尝试破坏方块时，会提示玩家并记录到日志中，方便服主追溯是谁在尝试破坏。
- 容器日志补充了原生容器打开/关闭事件，箱子、木桶、潜影盒以及其它可识别容器的打开与关闭记录更完整。

## 服务器公告

- 新增**进服弹窗公告**管理。入口：**服务器设置** → **进服弹窗公告**。
- 管理员最多可创建 5 条公告，可单独启用、停用、编辑、删除，并可预览当前启用公告。
- 玩家每次连接服务器时，会看到所有已启用公告组成的弹窗；公告内容支持颜色代码和 `\n` 换行，适合展示服务器规则、活动通知、维护提醒等。

## 经济系统

- 新增**死亡损失金币**功能，默认开启，玩家死亡时默认扣除 100 金币。
- 管理员可在 **服务器设置** → **经济系统管理** → **功能设置** 中配置是否启用死亡扣金币，以及每次死亡扣除的金币数量。
- 当玩家余额不足时，死亡惩罚最多扣到 0 金币，不会产生负数余额。

## 管理与排查工具

- **服务器实时面板**加入服务器设置首页，可查看 TPS、在线人数、实体数量、掉落物数量，以及经济、领地、日志、防刷、公会、PVP 等主要开关状态。
- **物品图标显示修复**加入经济系统管理。服务器启动时会自动检测其它附加包带来的自定义物品偏移，尽量修复商店、拍卖行、背包查看等界面的物品图标串位问题。
- 如果个别自定义物品仍显示为占位图标，管理员可在**物品图标显示修复**中手动指定该物品的贴图路径。

## PVP

- **PVP系统总开关**关闭时，插件会把互相伤害规则交回原版世界/存档设置处理，不再继续接管 PVP。
- PVP 管理中保留三种模式：**原版模式**、**插件模式**、**禁止模式**。服主可以按服务器玩法选择完全交给原版、使用插件规则与夺金统计，或强制禁止玩家互相伤害。

## 其他优化

- 观察实体的相机功能减少了跟随更新频率，降低持续观察时的脚本压力。
- 日志查询器已加入资源包贴图与物品名称，且不会被普通商店/拍卖上架流程当作可出售物品处理。
- 更新 Minecraft 脚本相关依赖到 **1.26.21 stable**，并精简行为包中不再需要声明的模块依赖。
