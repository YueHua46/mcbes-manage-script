# 2026-03-29 更新说明

## 【管理员】行为日志

- **玩家获得物品监控**：可登记若干物品的 type id；当任意玩家**获得**对应物品时，会写入行为日志，并**附带触发瞬间该玩家的背包快照**，便于排查刷物、异常来源等。入口：**行为日志管理** → **玩家获得物品监控**（可新增单项、清空全部等）。
- **如何填写物品 type id**：先**手持**目标物品，在聊天输入 **`/yuehua:get_item_typeid`** 可复制当前手持物品的 type id；若需查看背包内全部物品的 id，使用 **`/yuehua:get_item_typeid all`**。管理员亦可用 **`/yuehua:subscribe_item_hold`**（`add` / `remove` / `list` / `clear`）在命令行维护登记列表；生成蛋可一次性登记族群（如说明中的 `spawn_egg_group`）。

## 【管理员】防刷物品（收纳袋）

- **功能**：在总开关与子项均开启时，禁止将**收纳袋**（含染色收纳袋等 `*_bundle`）放入**非常规容器**（如漏斗、投掷器、发射器、熔炉类、酿造台、合成器等）；**木箱、陷阱箱、木桶、末影箱、潜影盒、铜箱系列**（type id 以 `copper_chest` 结尾）等视为常规箱子，**不拦截**。
- **总开关**：**功能开关管理**中的「防刷物品总开关」；**收纳袋子开关**、防刷白名单与方块白名单清理：**服务器设置** → **防刷物品设置**。
- **白名单**：**防刷白名单（玩家）**内的玩家放置受限容器方块时，会自动将该方块坐标记入**方块白名单**，该位置上的非常规容器不再按上述规则清除收纳袋；管理员不在名单则不会自动豁免。破坏方块会从坐标白名单中移除对应项。
- **性能与范围**：仅当容器**约 10 格内**存在玩家时才会扫描并纠正容器内的违规收纳袋；逻辑通过稳定的交互与物品栏事件驱动，**不依赖**部分环境下未启用的 beta 容器开关类 API。
